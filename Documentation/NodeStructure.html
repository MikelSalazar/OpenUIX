Node Class

Initially, to sustain the node structure the base Node class must contain the following public members: Name: A basic character string that unequivocally identifies the node among its peers. It must follow common naming rules: it should be an alphanumeric value (including uppercase and lowercase letters, numbers and hyphenation symbols) up to 256 characters, it must start with a letter and it must not be null. Type: Contains the metadata related to the node type, including its name, aliases, class constructor and extension properties (indicating if it an abstract class, the base class, etc.).

Node Hierarchy

Parent: A reference to the parent node instance. Necessary to navigate across the node hierarchy. Children: A collection of node instances Storing all child nodes into a single structure is useful for serialization (since it maintains the original order) but it poses other problems.
